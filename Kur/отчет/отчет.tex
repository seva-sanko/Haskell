	\section {Постановка задачи}
	
	Целью курсовой работы является разработка программного комплекса для обработки растровых изображений на языке Haskell. Работа разделена на две части:
	
	\begin{enumerate}
		\item \textbf{Часть 1:} Воспроизведение решения задачи обработки изображений, описанной в главах 12 и 13 книги Филиппа Хагенлохера «Haskell from the Very Beginning».
		Требуется:
		\begin{itemize}
			\item Реализовать парсер формата PNM (PPM/PGM) с использованием комбинаторных библиотек.
			\item Использовать обобщенные алгебраические типы данных (GADT) для типизации изображений.
			\item Реализовать параллельную обработку данных.
		\end{itemize}
		
		\item \textbf{Часть 2:} Разработка собственного синтаксического анализатора и графического редактора.
		Задачи:
		\begin{itemize}
			\item Разработать синтаксический анализатор файлов формата BMP (24-bit). Разобрать заголовок (File Header, DIB Header) и содержимое файла.
			\item Реализовать графические фильтры: инверсия, пороговое черно-белое преобразование, усиление резкости (матричная свертка), изменение яркости и контрастности, отражение по горизонтали и вертикали.
			\item Реализовать эффект «Глитч» (случайное смещение участков изображения).
			\item Реализовать функцию рирпроекции (хромакей): наложение изображения на фон с использованием маски по цвету (Green Screen / Blue Screen) с настраиваемым допуском.
			\item Организовать консольный пользовательский интерфейс (CLI) для взаимодействия с программой.
		\end{itemize}
	\end{enumerate}

	\section{Особенности реализации}
	Согласно заданию, решение разбито на два независимых проекта \texttt{stack}. Для работы с бинарными данными и обеспечения высокой производительности использовались библиотеки \texttt{bytestring}, \texttt{binary} и \texttt{vector}.
	
	\subsection{Часть 1: Реализация по книге Ф. Хагенлохера}
	
	В соответствии с главой 12 книги, для разбора текстовых и бинарных заголовков формата PNM была выбрана библиотека \texttt{attoparsec}, так как она обеспечивает высокую производительность и удобный интерфейс комбинаторов.
	
	В соответствии с главой 13, для представления данных использован подход GADT (Generalized Algebraic Data Types). Это позволяет определить единый тип \texttt{PNM}, конструкторы которого (\texttt{Greyscale} и \texttt{RGB}) явно кодируют формат данных, предотвращая ошибки типа ``Return-type polymorphism''.
	
	Для демонстрации возможностей многопоточности Haskell применена библиотека \texttt{parallel}. Реализована функция \texttt{processImageParallel}, которая разбивает вектор пикселей на фрагменты (чанки) и обрабатывает их параллельно на доступных ядрах процессора, используя стратегию \texttt{parList rdeepseq}.
	
	\subsection{Часть 2: Модификатор изображений BMP}
	
	\subsubsection{Типы данных}
	Для хранения изображения используется структура \texttt{Image}, содержащая ширину, высоту и плоский вектор байтов (\texttt{Word8}). Использование \texttt{Data.Vector.Storable} обеспечивает доступ к пикселям за время $O(1)$ и эффективное использование памяти, в отличие от списков.
	
	\begin{lstlisting}[caption={Определение типа Image}, label={lst:image_type}]
		module Types where
		
		import qualified Data.Vector.Storable as V
		import Data.Word (Word8)
		
		data Image = Image {
			imgWidth  :: Int,
			imgHeight :: Int,
			imgData   :: V.Vector Word8 
		} deriving (Show, Eq)
		
		data RGB = RGB { r :: Word8, g :: Word8, b :: Word8 } 
		deriving (Show, Eq)
	\end{lstlisting}
	
	\subsubsection{Синтаксический анализ BMP}
	Реализован собственный парсер формата BMP с использованием монады \texttt{Get} из библиотеки \texttt{Data.Binary}.
	
	Особенности реализации парсера:
	\begin{itemize}
		\item Чтение сигнатуры ``BM'' и заголовков (File Header, DIB Header).
		\item Проверка битности (поддержка 24-bit TrueColor).
		\item Учет выравнивания строк (Padding): каждая строка в BMP дополняется нулями до длины, кратной 4 байтам.
		\item Обработка порядка строк: BMP хранит данные снизу-вверх, поэтому при чтении строки разворачиваются.
	\end{itemize}
	
	\begin{lstlisting}[caption={Фрагмент парсера BMP}, label={lst:bmp_parser}]
		parseBMP :: Get Image
		parseBMP = do
		sig <- getWord16le -- "BM"
		skip 8 
		offset <- getWord32le
		headerSize <- getWord32le
		w <- getWord32le
		h <- getWord32le

		
		let rowSize = widthInt * 3
		let padding = (4 - (rowSize `mod` 4)) `mod` 4
		
		rawRows <- replicateM heightInt $ do
		row <- getByteString rowSize
		skip padding 
		return row
		
		let flatData = BS.concat (reverse rawRows) 
		return $ Image widthInt heightInt (V.fromList $ BS.unpack flatData)
	\end{lstlisting}    
	
	\subsubsection{Реализация фильтров}
	В модуле \texttt{Filters.hs} реализованы алгоритмы обработки изображений.
	
	\textbf{Пороговое черно-белое преобразование (Threshold):}
	Сравнение значения каждого канала с заданным порогом.
	
	\begin{lstlisting}[caption={Реализация порогового преобразования}, label={lst:bw}]
		thresholdBW :: Int -> Image -> Image
		thresholdBW level (Image w h v) = Image w h (V.map check v)
		where
		lvl = fromIntegral level
		check x = if x > lvl then 255 else 0
	\end{lstlisting}
	
	\textbf{Усиление резкости (Sharpen):}
	Реализовано через свертку (convolution) с матрицей ядра:
	$$
	\begin{pmatrix}
		0 & -1 & 0 \\
		-1 & 5 & -1 \\
		0 & -1 & 0 
	\end{pmatrix}
	$$
	Для каждого пикселя вычисляется взвешенная сумма его соседей.
	
	\textbf{Эффект «Глитч» (Glitch):}
	Алгоритм случайным образом выбирает два прямоугольных блока изображения и меняет их местами, создавая эффект цифрового искажения. Используется монада \texttt{IO} для генерации случайных координат.
	
	\subsubsection{Рирпроекция (Chroma Key)}
	Реализована функция наложения переднего плана (\texttt{fg}) на фон (\texttt{bg}).
	Алгоритм проверяет Евклидово расстояние между цветом текущего пикселя и ключевым цветом (хромакеем). Если расстояние меньше заданного допуска (\texttt{tolerance}), пиксель берется из фона, иначе — из переднего плана.
	
	\begin{lstlisting}[caption={Логика хромакея}, label={lst:chroma}]
		isKey :: (Word8, Word8, Word8) -> RGB -> Int -> Bool
		isKey (b, g, r) (RGB kr kg kb) tol =
		let dist = sqrt (db*db + dg*dg + dr*dr)
		in dist < fromIntegral tol
		
		chromaKey :: Image -> Image -> RGB -> Int -> Image
		chromaKey fg bg keyColor tol = ...

	\end{lstlisting}
	
	\subsubsection{Взаимодействие с пользователем}
	Реализован цикл обработки команд (\texttt{REPL}) в модуле \texttt{Main.hs}. Программа хранит текущее состояние (\texttt{AppState}), содержащее загруженное изображение. Поддерживаются команды загрузки, сохранения, применения эффектов и генерации тестовых изображений.

	\section*{Заключение}
	\addcontentsline{toc}{section}{Заключение}
	В рамках курсовой работы были разработаны два проекта на языке Haskell для обработки растровой графики.
	
	В первой части были освоены принципы комбинаторного парсинга (Attoparsec), использование GADT для строгой типизации и методы параллельного программирования для ускорения обработки изображений формата PNM.
	
	Во второй части был создан полноценный графический модификатор для формата BMP. Ключевые результаты:
	\begin{itemize}
		\item Написан собственный парсер бинарного формата BMP, корректно обрабатывающий заголовки и выравнивание.
		\item Использованы эффективные структуры данных (\texttt{Data.Vector.Storable}) для работы с пикселями, что обеспечило высокую скорость работы фильтров.
		\item Реализован широкий спектр алгоритмов: от простой инверсии до матричной свертки (резкость) и композитинга (хромакей).
		\item Создан удобный консольный интерфейс с поддержкой генерации тестовых данных.
	\end{itemize}
	
	Полученные навыки работы с бинарными данными, монадами состояния и векторами демонстрируют возможности Haskell в области системного программирования и обработки медиаданных.
