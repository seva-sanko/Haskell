	\section*{Введение}
	\addcontentsline{toc}{section}{Введение}
	
	Финансовая модель транзакций — это система учета и классификации денежных операций, которая позволяет анализировать финансовые потоки. В основе такой модели лежит формализованное представление транзакций с четко определенными атрибутами: идентификатором, суммой, типом операции и описанием.
	
	Отличительной особенностью финансовой модели  является разделение транзакций по типам операций. Доходы - это поступления денежных средств, расходы — затраты, а переводы — перемещения между счетами. 
	
	Транзакции могут быть классифицированы разными способами, использую механизмы фильтрации и разбиения на группы. Например, логика проверки пороговой стоимости позволяет выделять значимые средства. С другой стороны, при совместном использовании нескольких критериев возможна многомерная классификация транзакций, что позволяет получить максимум данных из информации и, соответственно, увеличить степень контроля.
	
	В данной работе представлена расширенная версия финансовой модели. Введены типы FinancialSummary для статистической сводки и TransactionMonitor a для расширенного анализа транзакций, и реализовали для них представителей указанных классов типов.

	\section{Особенности реализации}
	
	\subsection{Функторы и аппликативные функторы}
	
	Реализация Semigroup и Monoid для FinancialSummary:
	
	\begin{lstlisting}[caption={Определение типов данных для финансовых транзакций}, label={lst:types}]
	instance Semigroup FinancialSummary where
	(FinancialSummary inc1 exp1 cnt1 max1) <> (FinancialSummary inc2 exp2 cnt2 max2) =
	FinancialSummary
	{ totalIncome = inc1 + inc2
		, totalExpense = exp1 + exp2
		, transactionCount = cnt1 + cnt2
		, maxTransaction = case (max1, max2) of
		(Nothing, Nothing) -> Nothing
		(Just t1, Nothing) -> Just t1
		(Nothing, Just t2) -> Just t2
		(Just t1, Just t2) -> 
		if amount t1 >= amount t2 
		then Just t1 
		else Just t2
	}
	\end{lstlisting}
	
	Использование Maybe Transaction для представления максимальной транзакции позволяет корректно обрабатывать отсутствие данных. Нейтральный элемент mempty содержит нулевые значения и Nothing для максимальной транзакции.
	
	\subsection{Реализация TransactionMonitor как newtype}
	
	Функция \texttt{Конструктор}:
	
	\begin{lstlisting}[caption={Функция проверки порогового значения}, label={lst:threshold}]
	newtype TransactionMonitor a = TransactionMonitor 
	{ runMonitor :: Transaction -> (Bool, a) 
	}
	\end{lstlisting}
	
	Паттерн позволяет отделить логику мониторинга от её выполнения.
	
	\subsection{Реализация Functor для TransactionMonitor}
	
	\texttt{Интерфейс:} 
	
	\begin{lstlisting}[caption={Функция классификации транзакций}, label={lst:classify}]
	instance Functor TransactionMonitor where
	fmap f (TransactionMonitor monitor) = TransactionMonitor $ \t ->
	let (needsAttention, result) = monitor t
	in (needsAttention, f result)
	\end{lstlisting}
	
	Функция fmap применяет преобразование только к значению a, не изменяя флаг Bool.
	
	\subsection{Реализация Applicative для TransactionMonitor}
	
	Создает монитор, который никогда не требует внимания, а также возвращает константное значение для любой транзакции.
	
	\begin{lstlisting}[caption={Вспомогательные функции-предикаты}, label={lst:predicates}]
	pure x = TransactionMonitor $ \_ -> (False, x)
	\end{lstlisting}
	
	\textbf{<*> (аппликация):}
	\begin{lstlisting}[caption={Вспомогательные функции-предикаты}, label={lst:predicates}]
		(TransactionMonitor f) <*> (TransactionMonitor g) = 
		TransactionMonitor $ \t ->
		let (needsAttention1, func) = f t
		(needsAttention2, value) = g t
		in (needsAttention1 || needsAttention2, func value)
	\end{lstlisting}
	
	Это означает: если хотя бы один монитор требует внимания, результирующий монитор также требует внимания, при этом значения вычисляются независимо для каждого монитора.

	Разработанное решение служит практическим примером применения абстракций функционального программирования для создания систем обработки данных в финансовой области.

	\section*{Приложение. Исходный код}
	\addcontentsline{toc}{section}{Приложение. Исходный код}
	
	\begin{lstlisting}[caption={Полный исходный код программы}]
		{-# LANGUAGE GeneralizedNewtypeDeriving #-}
		
		module Lib where
		
		data TransactionType = Income | Expense | Transfer
		deriving (Show, Eq, Enum, Bounded)
		
		data Transaction = Transaction
		{ transactionId :: Int
			, amount :: Double
			, transactionType :: TransactionType
			, description :: String
			, account :: String 
		}
		deriving (Show, Eq)
		
		
		data FinancialSummary = FinancialSummary
		{ totalIncome :: Double
			, totalExpense :: Double
			, transactionCount :: Int
			, maxTransaction :: Maybe Transaction  
		}
		deriving (Show, Eq)
		
		
		instance Semigroup FinancialSummary where
		(FinancialSummary inc1 exp1 cnt1 max1) <> (FinancialSummary inc2 exp2 cnt2 max2) =
		FinancialSummary
		{ totalIncome = inc1 + inc2
			, totalExpense = exp1 + exp2
			, transactionCount = cnt1 + cnt2
			, maxTransaction = case (max1, max2) of
			(Nothing, Nothing) -> Nothing
			(Just t1, Nothing) -> Just t1
			(Nothing, Just t2) -> Just t2
			(Just t1, Just t2) -> 
			if amount t1 >= amount t2 
			then Just t1 
			else Just t2
		}
		
		instance Monoid FinancialSummary where
		mempty = FinancialSummary
		{ totalIncome = 0
			, totalExpense = 0
			, transactionCount = 0
			, maxTransaction = Nothing
		}
		
		newtype TransactionMonitor a = TransactionMonitor 
		{ runMonitor :: Transaction -> (Bool, a) 
		}
		
		instance Functor TransactionMonitor where
		fmap f (TransactionMonitor monitor) = TransactionMonitor $ \t ->
		let (needsAttention, result) = monitor t
		in (needsAttention, f result)
		
		instance Applicative TransactionMonitor where
		pure x = TransactionMonitor $ \_ -> (False, x)
		
		(TransactionMonitor f) <*> (TransactionMonitor g) = 
		TransactionMonitor $ \t ->
		let (needsAttention1, func) = f t
		(needsAttention2, value) = g t
		in (needsAttention1 || needsAttention2, func value)
		
		transactionToSummary :: Transaction -> FinancialSummary
		transactionToSummary t = FinancialSummary
		{ totalIncome = case transactionType t of Income -> amount t; _ -> 0
			, totalExpense = case transactionType t of Expense -> amount t; _ -> 0
			, transactionCount = 1
			, maxTransaction = Just t
		}
		
		transactionsToSummary :: [Transaction] -> FinancialSummary
		transactionsToSummary = foldMap transactionToSummary
		
		combinePeriodReports :: [FinancialSummary] -> FinancialSummary
		combinePeriodReports = mconcat  
		
		largeTransactionMonitor :: TransactionMonitor Double
		largeTransactionMonitor = TransactionMonitor $ \t ->
		let needsAttention = amount t > 1000
		in (needsAttention, amount t)
		
		suspiciousTransactionMonitor :: TransactionMonitor String
		suspiciousTransactionMonitor = TransactionMonitor $ \t ->
		let needsAttention = amount t < 0  
		in (needsAttention, description t)
		
		combinedMonitor :: TransactionMonitor (Double, String)
		combinedMonitor = (,) <$> largeTransactionMonitor <*> suspiciousTransactionMonitor
		
		generateNotificationDouble :: (Bool, Double) -> Maybe String
		generateNotificationDouble (needsAttention, amt)
		| needsAttention = Just $ " " ++ show amt
		| otherwise = Nothing
		
		generateNotificationString :: (Bool, String) -> Maybe String
		generateNotificationString (needsAttention, desc)
		| needsAttention = Just $ "'" ++ desc ++ "'"
		| otherwise = Nothing
		
		generateNotificationPair :: (Bool, (Double, String)) -> Maybe String
		generateNotificationPair (needsAttention, (amt, desc))
		| needsAttention = Just $ " '" ++ desc ++ "'  " ++ show amt
		| otherwise = Nothing
		
		complexMonitoringSystem :: Transaction -> [Maybe String]
		complexMonitoringSystem t =
		[ generateNotificationDouble $ runMonitor largeTransactionMonitor t
		, generateNotificationString $ runMonitor suspiciousTransactionMonitor t
		, generateNotificationPair $ runMonitor combinedMonitor t
		]
	\end{lstlisting}
